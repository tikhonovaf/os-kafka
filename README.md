# Задание 2: Система мониторинга с использованием Spring Kafka

Задание содержит два приложения, которые обеспечивают прием метрик из системы миграции виртуальных машин, запись метрик в БД и выборку метрик (всех метрик или для заданной вирткальной машины):
os-producer - для передачи метрик и выборки данных по метрикам, 
os-consumer - для записи метрик в БД

Сценарий работы

1. Вызовом rest сервиса POST /api/metric/vmMetrics (Добавление метрики) осуществляется отправка метрики в  Kafka топик "metrics-topic".
2. Приложение os-consumer принимает метрики из Kafka топика "metrics-topic" и записывает в БД
3. Для просмотра значений метрик можно использовать rest сервисы:
   GET   /api/metric/vmMetrics  Выборка метрик
   GET   /api/metric/vmMetrics/search  Выборка метрики по вирткальной машине

Вход в систему

      Для входа в систему нужно вызвать POST /api/login  Вход в систему и указать 
      {
        "username": "ADMIN",

База данных

      В приложении используется облачная БД из VK Cloud.  Настройка осуществлена в файле property.
